## Overview

`MoQSession` is the primary class that implements the MOQT (Media over QUIC Transport) protocol. It serves as both a **Publisher** and **Subscriber**, enabling bidirectional communication for media streaming applications.

**Header:** `moxygen/MoQSession.h`

### Key Features

- **Dual Role**: Implements both `Publisher` and `Subscriber` interfaces
- **Transport Agnostic**: Operates over any `proxygen::WebTransport` implementation
- **Extensible**: Applications set custom handlers via `setPublishHandler()` and `setSubscribeHandler()`
- **Coroutine-Based**: Uses folly coroutines for async operations
- **Version Negotiation**: Supports multiple MOQT protocol versions

### Inheritance

## Core Methods

### start

```cpp
void start();
```

Starts the session by initiating the control stream and beginning protocol operations. Must be called before any subscribe/publish operations.

### setup

```cpp
folly::coro::Task<ServerSetup> setup(ClientSetup setup);
```

Performs the MOQT handshake by sending a `CLIENT_SETUP` message and awaiting the `SERVER_SETUP` response.

<ParamField path="setup" type="ClientSetup" required>
  Client setup parameters including supported versions and role
</ParamField>

<ResponseField name="ServerSetup" type="folly::coro::Task<ServerSetup>">
  Server setup response containing negotiated version and parameters
</ResponseField>

**Usage:**

```cpp
ClientSetup clientSetup;
clientSetup.supportedVersions = {kVersionDraft07};
clientSetup.role = Role::BOTH;

auto serverSetup = co_await session->setup(std::move(clientSetup));
```

## Publisher Methods

These methods are inherited from the `Publisher` interface.

### subscribe

```cpp
folly::coro::Task<SubscribeResult> subscribe(
    SubscribeRequest sub,
    std::shared_ptr<TrackConsumer> callback
) override;
```

Sends a `SUBSCRIBE` request to the remote peer and awaits the response.

<ParamField path="sub" type="SubscribeRequest" required>
  Subscribe request containing track namespace, name, range, and parameters
</ParamField>

<ParamField path="callback" type="std::shared_ptr<TrackConsumer>" required>
  Consumer that will receive objects for this subscription
</ParamField>

<ResponseField name="SubscribeResult" type="folly::Expected<std::shared_ptr<SubscriptionHandle>, SubscribeError>">
  On success, returns a `SubscriptionHandle` for managing the subscription. On failure, returns `SubscribeError`.
</ResponseField>

**Usage:**

```cpp
SubscribeRequest subReq;
subReq.trackNamespace = {"conference", "audio"};
subReq.trackName = "main";
subReq.priority = 128;

auto consumer = std::make_shared<MyTrackConsumer>();
auto result = co_await session->subscribe(std::move(subReq), consumer);

if (result.hasValue()) {
  auto handle = result.value();
  // Use handle for unsubscribe or updates
}
```

## Subscriber Methods

These methods are inherited from the `Subscriber` interface.

### publish

```cpp
PublishResult publish(
    PublishRequest pub,
    std::shared_ptr<Publisher::SubscriptionHandle> handle = nullptr
) override;
```

Initiates a `PUBLISH` request, allowing the local application to publish a track to the remote peer.

<ParamField path="pub" type="PublishRequest" required>
  Publish request containing track namespace, name, and parameters
</ParamField>

<ParamField path="handle" type="std::shared_ptr<Publisher::SubscriptionHandle>">
  Optional subscription handle if this is a forwarded publish
</ParamField>

<ResponseField name="PublishResult" type="folly::Expected<PublishConsumerAndReplyTask, PublishError>">
  On success, returns a `TrackConsumer` to write objects and an async reply task. On failure, returns `PublishError`.
</ResponseField>

**Usage:**

```cpp
PublishRequest pubReq;
pubReq.trackNamespace = {"conference", "video"};
pubReq.trackName = "camera-1";

auto result = session->publish(std::move(pubReq));
if (result.hasValue()) {
  auto& [consumer, replyTask] = result.value();
  
  // Start publishing objects immediately
  auto subgroup = consumer->beginSubgroup(groupId, subgroupId, priority);
  subgroup->object(objectId, payload, extensions, false);
  
  // Await the PUBLISH_OK response
  auto reply = co_await std::move(replyTask);
}
```

## Example Usage

### Client Example

```cpp
// Create session
auto wt = /* obtain WebTransport */;
auto exec = std::make_shared<MoQExecutor>();
auto session = std::make_shared<MoQSession>(wt, exec);

// Configure
session->setVersion(kVersionDraft07);

// Start and setup
session->start();
ClientSetup clientSetup{.supportedVersions = {kVersionDraft07}};
auto serverSetup = co_await session->setup(std::move(clientSetup));

// Subscribe to a track
SubscribeRequest subReq;
subReq.trackNamespace = {"conference"};
subReq.trackName = "audio";

auto consumer = std::make_shared<MyTrackConsumer>();
auto result = co_await session->subscribe(std::move(subReq), consumer);
```

### Server Example

```cpp
class MyServerSetup : public MoQSession::ServerSetupCallback {
  folly::Try<ServerSetup> onClientSetup(
      ClientSetup clientSetup,
      const std::shared_ptr<MoQSession>& session) override {
    ServerSetup serverSetup;
    serverSetup.selectedVersion = kVersionDraft07;
    return folly::Try<ServerSetup>(std::move(serverSetup));
  }
  
  folly::Expected<folly::Unit, SessionCloseErrorCode> validateAuthority(
      const ClientSetup& clientSetup,
      uint64_t negotiatedVersion,
      std::shared_ptr<MoQSession> session) override {
    return folly::unit;
  }
};

MyServerSetup setupCallback;
auto session = std::make_shared<MoQSession>(wt, setupCallback, exec);
session->setPublishHandler(myPublisher);
session->start();
```
