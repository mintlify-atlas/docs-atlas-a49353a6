## Overview

`MoQCache` stores published objects in memory and serves them to new subscribers requesting historical data. It integrates with `MoQRelay` to provide caching capabilities and supports FETCH requests with intelligent upstream coalescing.

**Source**: `moxygen/relay/MoQCache.h`

## Core Methods

### getSubscribeWriteback

Returns a filter that caches objects as they're received from upstream.

```cpp
std::shared_ptr<TrackConsumer> getSubscribeWriteback(
    const FullTrackName& ftn,
    std::shared_ptr<TrackConsumer> consumer);
```

<ParamField path="ftn" type="const FullTrackName&">
  The track being subscribed to
</ParamField>

<ParamField path="consumer" type="std::shared_ptr<TrackConsumer>">
  The downstream consumer to pass objects to
</ParamField>

<ResponseField name="return" type="std::shared_ptr<TrackConsumer>">
  A wrapped consumer that caches objects before forwarding
</ResponseField>

**Usage in relay**:
```cpp
auto cachedConsumer = cache_->getSubscribeWriteback(ftn, consumer);
co_await upstream->subscribe(subReq, cachedConsumer);
```

### fetch

Serves a FETCH request from cache, fetching missing ranges from upstream.

```cpp
folly::coro::Task<Publisher::FetchResult> fetch(
    Fetch fetch,
    std::shared_ptr<FetchConsumer> consumer,
    std::shared_ptr<Publisher> upstream);
```

<ParamField path="fetch" type="Fetch">
  The fetch request specifying range to retrieve
</ParamField>

<ParamField path="consumer" type="std::shared_ptr<FetchConsumer>">
  Consumer to receive cached and fetched objects
</ParamField>

<ParamField path="upstream" type="std::shared_ptr<Publisher>">
  Upstream publisher to fetch missing objects from
</ParamField>

**Behavior**:
1. Serves cached objects in the requested range immediately
2. Identifies gaps (missing objects)
3. Issues upstream FETCH for missing ranges
4. Caches fetched objects for future requests
5. Coalesces concurrent fetches for the same range


## Cache Entry Structure

### CacheEntry

```cpp
struct CacheEntry {
  uint64_t subgroup;
  ObjectStatus status;              // Normal, EndOfGroup, EndOfTrackAndGroup
  Extensions extensions;
  Payload payload;
  bool complete;                    // Fully received (not partial)
  bool forwardingPreferenceIsDatagram;
  TimePoint cachedAt;              // For time-based expiry
};
```

### CacheGroup

```cpp
struct CacheGroup {
  folly::F14FastMap<uint64_t, std::unique_ptr<CacheEntry>> objects;
  uint64_t maxCachedObject;
  bool endOfGroup;
  std::optional<uint64_t> seenPriorGroupIdGap;
};
```

### CacheTrack

```cpp
struct CacheTrack {
  folly::F14FastMap<uint64_t, std::shared_ptr<CacheGroup>> groups;
  bool isLive;                      // Currently receiving data
  bool endOfTrack;
  std::optional<AbsoluteLocation> largestGroupAndObject;
  FetchInProgressSet fetchInProgress; // Coalesces concurrent fetches
  std::list<uint64_t> groupLRU;    // LRU list for groups
  std::optional<std::chrono::milliseconds> maxCacheDuration;
  Extensions extensions;            // Track-level extensions
};
```

## LRU Eviction

The cache uses a two-level LRU:

1. **Track-level LRU**: Evicts entire tracks when `maxCachedTracks` is exceeded
2. **Group-level LRU**: Evicts oldest groups per track when `maxCachedGroupsPerTrack` is exceeded

**Evictability rules**:
- Tracks are evictable when not live and have no active fetches
- Groups are evictable when not part of an active fetch interval

```cpp
bool CacheTrack::canEvict() const {
  return !isLive && activeFetchCount == 0;
}
```

## Fetch Coalescing

When multiple subscribers issue concurrent FETCHes for overlapping ranges:

1. The cache maintains a `FetchInProgressSet` per track
2. New fetches check for overlapping in-progress fetches
3. If overlap exists, the new fetch waits for the in-progress fetch to complete
4. Only one upstream FETCH is issued per unique range

## Usage Example

```cpp cache_integration.cpp
#include <moxygen/relay/MoQCache.h>

auto cache = std::make_unique<MoQCache>(
  /* maxCachedTracks */ 50,
  /* maxCachedGroupsPerTrack */ 5
);

// Configure time-based expiry
cache->setMaxCacheDuration(
  FullTrackName{"live-stream", "video"},
  std::chrono::minutes(5)
);

// Wrap upstream subscription with caching
auto cachedConsumer = cache->getSubscribeWriteback(ftn, consumer);
co_await upstreamPublisher->subscribe(subReq, cachedConsumer);

// Handle FETCH requests
co_await cache->fetch(fetchReq, fetchConsumer, upstreamPublisher);

// Query cache state
if (cache->hasTrack(ftn)) {
  if (cache->hasCachedObject(ftn, AbsoluteLocation{groupID, objectID})) {
    // Object is cached
  }
}
```
