## Overview

`MoQServer` is the server-side class for accepting MOQT connections. It supports both HTTP/3 WebTransport and native QUIC connections, handling incoming connections and creating `MoQSession` instances for each client.

**Header:** `moxygen/MoQServer.h`

**Namespace:** `moxygen`

## Usage Examples

### Basic Server Setup

```cpp
#include <moxygen/MoQServer.h>
#include <folly/init/Init.h>

using namespace moxygen;

int main(int argc, char* argv[]) {
  folly::Init init(&argc, &argv, true);
  
  // Create server
  auto server = std::make_shared<MoQServer>(
      "cert.pem",      // Certificate file
      "key.pem",       // Private key file
      "/moq"           // WebTransport endpoint
  );
  
  // Start listening
  folly::SocketAddress addr("0.0.0.0", 4433);
  server->start(addr);
  
  // Run event loop
  folly::EventBaseManager::get()->getEventBase()->loopForever();
  
  return 0;
}
```

### Relay Server with Custom Handlers

```cpp
class MoQRelayServer : public MoQServer {
 public:
  MoQRelayServer(const std::string& cert, const std::string& key)
      : MoQServer(cert, key, "/moq-relay") {}

  void onNewSession(std::shared_ptr<MoQSession> clientSession) override {
    // Set handlers for each new session
    clientSession->setPublishHandler(relay_);
    clientSession->setSubscribeHandler(relay_);
  }

 protected:
  std::shared_ptr<MoQSession> createSession(
      folly::MaybeManagedPtr<proxygen::WebTransport> wt,
      std::shared_ptr<MoQExecutor> executor) override {
    return std::make_shared<MoQRelaySession>(
        std::move(wt),
        *this,
        std::move(executor));
  }

 private:
  std::shared_ptr<MoQRelay> relay_{std::make_shared<MoQRelay>()};
};

int main(int argc, char* argv[]) {
  folly::Init init(&argc, &argv, true);
  
  auto server = std::make_shared<MoQRelayServer>("cert.pem", "key.pem");
  server->start(folly::SocketAddress("0.0.0.0", 9668));
  
  folly::EventBaseManager::get()->getEventBase()->loopForever();
  return 0;
}
```

```cpp
class DatePublisher : public Publisher {
 public:
  folly::coro::Task<SubscribeResult> subscribe(
      SubscribeRequest subReq,
      std::shared_ptr<TrackConsumer> consumer) override {
    
    if (subReq.fullTrackName.trackName != "date") {
      co_return folly::makeUnexpected(SubscribeError{
          subReq.requestID,
          SubscribeErrorCode::TRACK_NOT_EXIST,
          "Track not found"});
    }
    
    // Set track alias
    consumer->setTrackAlias(TrackAlias(subReq.requestID.value));
    
    // Add subscriber and start publishing
    auto session = MoQSession::getRequestSession();
    addSubscriber(session, subReq, consumer);
    
    co_return SubscribeOk{
        .requestID = subReq.requestID,
        .expires = std::chrono::milliseconds(0),
        .largest = currentLocation()};
  }
};

  DateServer(const std::string& cert, const std::string& key)
      : MoQServer(cert, key, "/moq-date"),
        publisher_(std::make_shared<DatePublisher>()) {}

  void onNewSession(std::shared_ptr<MoQSession> session) override {
    session->setPublishHandler(publisher_);
  }

 private:
  std::shared_ptr<DatePublisher> publisher_;
```

## Transport Configuration

### Default Transport Settings

When `transportSettings` is not provided, `MoQServer` uses these defaults:

```cpp
quic::TransportSettings defaults;

// Congestion control
defaults.defaultCongestionController = quic::CongestionControlType::Copa;
defaults.copaDeltaParam = 0.05;
defaults.pacingEnabled = true;
defaults.maxCwndInMss = quic::kLargeMaxCwndInMss;

// Batching and performance
defaults.batchingMode = quic::QuicBatchingMode::BATCHING_MODE_GSO;
defaults.maxBatchSize = 48;
defaults.dataPathType = quic::DataPathType::ContinuousMemory;
defaults.maxServerRecvPacketsPerLoop = 10;
defaults.writeConnectionDataPacketsLimit = 48;

// Flow control windows (1 MB each)
defaults.advertisedInitialConnectionFlowControlWindow = 1024 * 1024;
defaults.advertisedInitialBidiLocalStreamFlowControlWindow = 1024 * 1024;
defaults.advertisedInitialBidiRemoteStreamFlowControlWindow = 1024 * 1024;
defaults.advertisedInitialUniStreamFlowControlWindow = 1024 * 1024;
```

### Custom Transport Settings

```cpp
quic::TransportSettings customSettings;

// Larger flow control windows for high-bandwidth scenarios
```

## Multi-Threaded Operation

`MoQServer` supports multi-threaded operation where each worker thread handles a subset of connections:

```cpp
const size_t numThreads = 4;
std::vector<std::thread> threads;
std::vector<folly::EventBase*> evbs;

// Create worker threads
for (size_t i = 0; i < numThreads; ++i) {
  threads.emplace_back([&evbs]() {
    folly::EventBase evb;
    evbs.push_back(&evb);
    evb.loopForever();
  });
}

// Wait for event bases to be ready
while (evbs.size() < numThreads) {
  std::this_thread::sleep_for(std::chrono::milliseconds(10));
}

// Start server with worker threads
server->start(folly::SocketAddress("0.0.0.0", 4433), evbs);

// Join threads on shutdown
for (auto& thread : threads) {
  thread.join();
}
```

<Note>
  Each `MoQSession` created by the server is bound to a single worker thread's 
  event base and MUST only be accessed from that thread.
</Note>

## Connection Lifecycle

The server handles two types of connections:

### HTTP/3 WebTransport Connections

1. Client sends HTTP CONNECT request to the configured endpoint
2. Server validates the request path matches the endpoint
3. Server responds with 200 OK and WebTransport headers
4. Server creates `MoQSession` with the WebTransport instance
5. `onNewSession()` is called for application setup

### Native QUIC Connections

1. Client establishes QUIC connection with MOQT ALPN
2. Server accepts connection if ALPN matches
3. Server wraps QUIC socket in `QuicWebTransport`
4. Server creates `MoQSession` with the transport
5. `onNewSession()` is called for application setup

## Takeover Support

`MoQServer` supports zero-downtime restarts via connection takeover:

```cpp
// Old process: Allow being taken over
server->allowBeingTakenOver(folly::SocketAddress("127.0.0.1", 9999));
int takeoverFD = server->getTakeoverHandlerSocketFD();
std::vector<int> socketFDs = server->getAllListeningSocketFDs();

// New process: Take over connections
server->setListeningFDs(socketFDs);
server->setProcessId(newProcessId);
server->waitUntilInitialized();
server->startPacketForwarding(folly::SocketAddress("127.0.0.1", 9999));

// Old process: Wind down
server->rejectNewConnections([]() { return true; });
server->pauseRead();
server->stop();
```
